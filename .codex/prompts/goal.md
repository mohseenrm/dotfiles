Set, view, pause, resume, or clear a session Goal — a persistent objective with
evidence-gated completion.

A Goal is an objective you hold for the rest of this session. Once set:
- Keep it in view across turns; let it steer planning and prioritization.
- Only consider it complete after verifying against concrete evidence (test
  output, benchmark numbers, a built artifact, command output, inspected files).
  Never mark it done because the work is "probably" finished.
- If the objective is not yet satisfied, take the next best action toward it
  rather than declaring completion.
