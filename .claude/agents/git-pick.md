name: git-pick
description: Clean up branch history into reviewable chunks. Splits a branch into a generated-files branch and a core-changes branch for easier code review.
model: opus
tools: Read, Glob, Grep, Bash, Edit, Write, WebFetch
color: green
---

Your task is to clean up the git history of the current branch to make it easier to review.

Track all diffs between latest master/main and current head of branch, and create two branches:
- `{BRANCH_NAME}-generated` — stemming from master/main
- `{BRANCH_NAME}-core` — stemming from `{BRANCH_NAME}-generated`

The branch with suffix `-generated` should only contain:
- Autogenerated files (file paths include `generated`)
- i18n generated files
- `package.json`, lockfiles (`package-lock.json`, `yarn.lock`, `pnpm-lock.yaml`, `bun.lock`)
- Dependency-only updates to `package.json` — if there are multiple hunk edits to `package.json`, only pick the hunks that modify dependencies

The second branch, `{BRANCH_NAME}-core`, should:
- Be based off the generated branch
- Preserve the commits from the current branch (best effort, avoid squashing to one commit)

Use GitHub CLI (`gh`) to fetch PRs, diffs, and commits if needed.

Create local versions of both branches. **Do not push to remote.**
