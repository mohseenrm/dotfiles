## Git Commit Guidelines

### Commit Message Format

Use these prefixes:

Feat: New feature

- Fix: Bug fix
- Refactor: Code refactoring
- Style: Formatting/styling changes
- Docs: Documentation updates
- Build: Maintenance tasks
- Test: Test-related changes

For Monorepose, add context as to which part of the code base this is modifying. Keep commit messages short and consice.

Example

```
Feat(dashboard): Adding new widget
```

### Working Process

1. Understand First

- Read existing code and documentation
- Understand project structure before making changes
- Ask clarifying questions if needed

2. Make Changes Incrementally

- Write code in logical chunks
- Commit after each meaningful change
- Keep commits atomic and focused
- Commit messages should be short and consice (< 50 chars)
